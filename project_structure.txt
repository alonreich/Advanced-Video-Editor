# This file maps each Python file in the project to its roles and duties.

`advanced_video_editor.py`:
- The main entry point. Sets up logging, exception handling, and the Qt environment.
- Patches the `mpv` DLLs if needed via `BinaryManager` and launches `MainWindow`.

`asset_loader.py`:
- Manages the import process for media files.
- Offloads probing (`ProbeWorker`) and thumbnail generation (`ThumbnailWorker`) to background threads to keep the UI responsive.
- Creates `ClipModel` instances from imported files and adds them to the timeline.

`binary_manager.py`:
- Static utility to manage external binary paths (ffmpeg, ffprobe, mpv).
- Handles the auto-patching of `libmpv-2.dll` to `mpv-1.dll` on Windows.

`clip_item.py`:
- The visual `QGraphicsRectItem` representing a clip on the timeline.
- Handles user input for selection, trimming, moving, and fade adjustments.
- Offloads the actual painting logic to `clip_painter.py`.

`clip_manager.py`:
- Handles logic for clip operations: splitting, deleting, and parameter updates.
- Acts as a middleman between the UI and the data model to ensure undo states are saved before modifications.

`clip_painter.py`:
- Static class responsible for the raw QPainter commands to draw clips.
- Draws backgrounds, waveforms, thumbnails, and fade gradients.

`export_dialog.py`:
- The UI dialog for rendering the project.
- Configures export settings and launches the `RenderWorker`.

`ffmpeg_generator.py`:
- The core engine for building FFmpeg filter graphs.
- Translates the timeline state (clips, cuts, tracks, effects, volumes) into a complex FFmpeg command string used for both playback and export.

`history.py`:
- Implements `UndoStack` with smart delta compression.
- Calculates differences between states to minimize memory usage ("Smart Push").

`inspector.py`:
- The properties panel. Displays controls for the selected clip (Speed, Volume, Crop, etc.).
- Manages global project settings like Output Resolution.

`main_window.py`:
- The central hub that instantiates all docks and controllers.
- Delegates specific logic to `ProjectController`, `ClipManager`, and `PlaybackManager`.
- Connects signals between the Timeline, Preview, and Inspector.

`media_pool.py`:
- A drag-and-drop enabled list widget for managing imported source files.

`model.py`:
- Defines the `ClipModel` dataclass.
- Stores pure data for a clip (path, start, duration, visual properties) separate from the UI code.

`playback_manager.py`:
- Manages the playback state loop and the synchronization between the `MPVPlayer` and the Timeline playhead.
- Detects when the timeline is "dirty" and triggers a filter graph rebuild via `ffmpeg_generator.py`.

`player.py`:
- Wraps the `libmpv` instance in a Qt widget.
- Provides methods for seeking, pausing, and applying live video filters (like crop).

`preview.py`:
- Container for the video player and overlay controls.
- `SafeOverlay`: Draws interactive handles for transforming (scale/move) and cropping clips directly on the video preview.
- `PreviewWidget`: Handles play/pause buttons and pop-out window logic.

`prober.py`:
- `ProbeWorker`: Runs `ffprobe` to extract metadata from new media.
- `WaveformWorker`: Runs `ffmpeg` to generate audio visualization images.

`project.py`:
- Low-level file system operations for projects.
- Handles creating project directories, saving JSON files, and managing the `assets` folder.

`project_controller.py`:
- High-level logic for Project actions: New, Open, Save As, and Autosave.
- restoring UI states (zoom level, scroll position) when loading a project.

`render_worker.py`:
- A background thread that executes the final FFmpeg export command to prevent UI freezing.

`system.py`:
- Logging configuration and a thread-safe `ConfigManager` for saving application preferences (layout, geometry).

`timeline_container.py`:
- The structural container that holds `TrackHeaders` and `TimelineView` side-by-side.

`timeline_scene.py`:
- The graphics scene for the timeline. Draws the background grid and track lines.

`timeline_view.py`:
- The interactive view for the timeline.
- Handles complex mouse events for the Playhead, Rubberband selection, and Razor tool.
- Manages zooming and scrolling logic.

`track_header.py`:
- UI for individual track controls (Names, Volume).
- Implements drag-and-drop logic for reordering tracks.

`worker.py`:
- `ThumbnailWorker`: Generates start/end thumbnails for video clips in a queue.
